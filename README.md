# 🗡️ Sanskar's NixOS Architecture

A comprehensively designed, highly optimized, and fully declarative NixOS configuration. This repository utilizes **Nix Flakes** and **Home Manager** to create a unified system that is simultaneously a robust gaming machine, an elite development workstation, and an AI-integrated powerhouse.

## ⚡ Core System & Hardware Optimizations

### Boot Sequence (Zero-Latency)
The bootloader has been heavily stripped down to eliminate all hesitation.
- **Bootloader**: Driven by `systemd-boot` configured with a **0-second timeout** for instant hand-off to the OS. (Hold `SPACE` during BIOS splash to access the boot menu for recovery).
- **Initrd**: Migrated from legacy bash initrd to **systemd-initrd**, allowing parallel loading of system modules during early boot.
- **Filesystem**: The root and home `ext4` partitions are mounted with `noatime` to eliminate disk I/O overhead generated by file read timestamps.
- **Console**: Verbosity is suppressed (`quiet`, `consoleLogLevel = 0`) during normal operation but strictly preserves critical error logging and an emergency shell fallback (`boot.shell_on_fail`) if hardware initialization fails.

### Graphics & Hybrid Architecture
Tuned specifically for a laptop utilizing an **AMD Renoir CPU + NVIDIA RTX 3050 Mobile** hybrid layout.
- **NVIDIA PRIME**: Configured strictly in `offload` mode. The AMD integrated GPU handles the KDE desktop and browser rendering to maximize battery life, while the NVIDIA GPU is spun up dynamically only when explicitly requested (e.g., launching a game).
- **Driver Stability**: Locked to the `production` proprietary NVIDIA driver branch on kernel `v6.12` to prevent Wayland black-screen regressions common on bleeding-edge driver releases.
- **Driver Blacklisting**: Aggressively blacklists `nouveau`, `mt7921e`, and `rtw88_8822bu` to force the system to use the compiled external `rtl88x2bu` WiFi driver.

## 💻 The Developer Environment

User space is entirely managed by **Home Manager** (`home/sanskar/`), treating the CLI as a first-class citizen.

### Modern Shell & Navigation
The standard GNU utilities have been entirely replaced with modern, heavily-threaded Rust/Go alternatives.
- **Shell**: `zsh` augmented with `starship` cross-shell prompt, syntax highlighting, and `carapace` providing completions for over 1000 tools.
- **Terminal Multiplexer**: `tmux` configured with vim-style pane navigation, horizontal/vertical splits (`-` and `|`), and a custom Dracula status bar.
- **Core Utilities**: 
  - `eza` (replaces `ls` with tree-views and git integration)
  - `bat` (replaces `cat` with syntax highlighting and git-gutters)
  - `zoxide` (replaces `cd` with an intelligent learning algorithm)
  - `ripgrep` & `fd` (ultra-fast search)
  - `yazi` (blistering fast asynchronous terminal file manager)
- **History Management**: Replaced standard history with `atuin`, providing a globally synced, SQLite-backed fuzzy-searchable shell history (Ctrl+R).
- **Git Engine**: Integrates `lazygit` for terminal UI version control, paired with `delta` for side-by-side syntax-highlighted diffs in the terminal.

### Hybrid Neovim (LazyVim) Architecture
The Neovim setup bridges the gap between Nix's immutable nature and the fluid plugin ecosystems of Neovim.
- **Nix Responsibilities**: `home/sanskar/nvim.nix` injects all heavy system dependencies directly into Neovim's path. This includes compilers (`gcc`, `gnumake`), formatters (`black`, `stylua`, `prettier`, `nixfmt-rfc-style`), Linters (`shellcheck`), and LSP servers (`pyright`, `ruff`, `zls`, `nimlangserver`, `nil`).
- **Lua Responsibilities**: Instead of managing Neovim plugins via Nixpkgs, the configuration symlinks a standard `lua/` directory into `~/.config/nvim`. This allows the **Lazy.nvim** package manager to download, compile, and cache plugins natively, resulting in sub-30ms startup times without fighting read-only file system errors.

### Supported Language Stacks
- **Python**: `python313`, `uv` (fast package manager), `ruff`, `debugpy` (DAP).
- **Zig & Nim**: System compilers, `zls`, and `nimlangserver`.
- **Common Lisp**: `sbcl` and `roswell`.
- **Web**: `nodejs_22`, `bun`, `typescript-language-server`.

## 🎮 The "Double-Click .exe" Gaming Engine

Gaming on this system is fully automated. The user does not need to configure Wine prefixes manually.

1. **MIME Hijacking**: `home/sanskar/default.nix` intercepts any interactions with `application/x-ms-dos-executable` or `.exe` files and routes them to a custom desktop application handler.
2. **The Wrapper Script**: The handler invokes `scripts/wine-run.sh`. 
3. **Prefix Provisioning**: When you double-click `setup.exe` for a game, the script evaluates the directory, generates an isolated `.wine-prefix` folder exclusively for that game, and quietly bootstraps it using `winetricks` (injecting `dxvk`, `vcrun2022`, and corefonts).
4. **Execution**: It launches the `.exe` via `wineWowPackages.stagingFull`, piped through `gamemode` for CPU prioritization and `gamescope` for resolution scaling.

### Anti-Crash Fixes for Repacks
FitGirl repacks use an aggressive multi-threaded Oodle decompressor that instantly crashes standard Linux kernels due to memory mapping limits.
- **Kernel Fix**: `boot.kernel.sysctl."vm.max_map_count" = 2147483642;`
- **PAM Limits**: Implements a hard system override granting `unlimited` stack sizes to user processes, preventing Wine from throwing stack overflow exceptions during installation phases.

## 🧠 Local AI & Virtualization

- **AI Tools**: Globally installs cutting-edge developer AI tools directly from the `unstable` branch, including `codex`, `gemini-cli`, `qwen-code`, and `opencode`.
- **LM Studio**: A custom system symlink maps LM Studio's hidden internal model database directly to `~/models`, allowing the user to manage GGUF models easily in their home directory.
- **Virtualization**: Supports `docker` (with custom DNS), `libvirtd` (with TPM emulation to support Windows 11 VMs), and `distrobox` for running isolated containers of Ubuntu/Fedora directly inside the terminal.

## 🛠️ Repository & System Management

### Daily Usage Aliases
The `shell.nix` module provides critical abstractions over the Nix CLI to manage the system safely.

| Alias | Command | Purpose |
|-------|---------|---------|
| `nrs` | `sudo nixos-rebuild switch` | Hot-swaps the system configuration. |
| `nrb` | `sudo nixos-rebuild boot` | Safely compiles the OS but delays activation until reboot (prevents NVIDIA crashes). |
| `nup` | `nix flake update && nrs` | Pulls the latest package revisions from GitHub and rebuilds. |
| `ns`  | `ns <pkg>` | Uses `fzf` and `jq` to fuzzy-search the stable Nixpkgs repository natively in the terminal. |
| `nu`  | `nu <pkg>` | Fuzzy-searches the bleeding-edge `unstable` repository. |
| `nclean` | `nix-collect-garbage -d` | Purges old system generations to reclaim disk space. |

### Directory Structure
```text
.
├── flake.nix                  # Flake entry point (Inputs, Overlays, HM Injection)
├── hosts/nixos/               # Hardware and system-wide configurations
│   ├── default.nix            # Core Nix settings, Kernel, Mirrors, Network
│   ├── hardware.nix           # File system mounts (noatime optimized)
│   └── kanata.kbd             # Global keyboard remapping logic
├── modules/                   # Reusable NixOS building blocks
│   ├── boot.nix               # systemd-boot, initrd, and kernel verbosity
│   ├── desktop.nix            # SDDM, KDE Plasma 6, PipeWire
│   ├── gaming.nix             # Wine, Gamescope, sysctl memory fixes
│   ├── nvidia.nix             # Proprietary drivers, PRIME Offload definitions
│   └── virtualization.nix     # Docker, Libvirtd, QEMU/KVM
├── home/sanskar/              # Userspace configurations (Home Manager)
│   ├── default.nix            # HM Entry point, MIME handling
│   ├── shell.nix              # ZSH, Starship, Aliases, functions
│   ├── tools.nix              # CLI replacements (bat, eza, tmux, atuin)
│   ├── git.nix                # Gitconfig, delta (diff viewer), gh CLI
│   ├── dev.nix                # Compilers, debuggers, languages
│   ├── nvim.nix               # System-level Neovim injected dependencies
│   └── nvim/                  # LazyVim Lua configuration mapping
└── scripts/                   # Standalone bash utilities
```
